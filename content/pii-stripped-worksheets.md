---

### ICP Red Flags

Watch for these during facilitation:

- "We can help anyone" -- push for specificity on the BEST fit
- "Both enterprise and SMB" -- pick ONE to start
- No clear trigger event defined -- they don't know when buyers become buyers
- Can't name a real person who fits -- ICP is theoretical, not based on evidence
- Targeting based on demographics not problems -- flip to problem-first thinking

### AI Prompt for ICP Refinement

Use with Claude or Google AI Studio to help refine the ICP:

```
I am helping a startup define their Ideal Customer Profile. Here is what we know:

Company: [Name]
Product/Service: [What they sell]
Current customers: [Any existing customers]
Target market: [Who they think they are targeting]

Help me refine their ICP by:
1. Suggesting a specific persona with name, role, and company type
2. Identifying 5 key pain points this persona experiences
3. Writing a Jobs-to-be-Done statement
4. Identifying 3 trigger events that would make them buy
5. Listing 3 common objections or barriers

Make it specific and actionable.
```

## 3. ICP Validation Questions

Five questions for pressure-testing any ICP statement. Can be used as a quick verbal checklist during facilitation.

1. Who is your single best existing customer or prospect? What makes them the best?
2. Who should you NOT target? What makes someone a bad fit?
3. When this person has this problem, how do they currently find solutions?
4. What would make them say no to you immediately?
5. Who else is involved in the buying decision? How long does it typically take?

## 4. Opportunity Segmentation Matrix

Use this matrix to help a founder evaluate 2-4 customer segments before committing to a primary ICP.

| Segment | Description | Pain Intensity (1-5) | Willingness to Pay (1-5) | Accessibility (1-5) | Future Potential (1-5) | Total |
|---------|-------------|---------------------|--------------------------|---------------------|------------------------|-------|
| | | | | | | |
| | | | | | | |
| | | | | | | |
| | | | | | | |

**Scoring Guide:**
- Pain Intensity: How much does this problem hurt them? (5 = urgent, keeps them up at night)
- Willingness to Pay: Would they pay for a solution? How much? (5 = budget exists, decision is easy)
- Accessibility: Can you reach them easily? (5 = direct access, no gatekeepers)
- Future Potential: Does this segment grow over time? (5 = fast-growing, expanding TAM)

**Highest total score = best starting ICP.** Not always, but usually.

## 5. ICP Red Flags

Quick reference for facilitators during Session 1.1.

| Red Flag | What to Say |
|----------|------------|
| "We can help anyone" | "Who suffers most? Start there." |
| "Both enterprise and SMB" | "Which one pays faster and needs less convincing?" |
| No trigger event defined | "When exactly does someone start looking for this?" |
| Can't name a real person | "Name one real human who would be a perfect customer." |
| Features, not problems | "Forget what you build -- what problem does this person have?" |
| ICP based on industry only | "What specific situation inside that industry triggers the need?" |

## 7. Positioning Statement Template

**Format:**

> "For [target customer] who [situation or need], [your company] is a [category] that [key benefit]. Unlike [competitors or alternatives], we [key differentiator]."

**Your Statement:**

> "For [_______] who [_______], [_______] is a [_______] that [_______]. Unlike [_______], we [_______]."

**Example of a well-formed positioning statement (generic):**

> "For [operations managers at mid-size e-commerce companies] who [are drowning in manual reporting and spreadsheet reconciliation], [Company Name] is a [workflow automation platform] that [cuts reporting time by 80% and surfaces inventory issues before they become stockouts]. Unlike [generic business intelligence tools that require a data analyst to configure], we [connect directly to their existing e-commerce stack and generate reports automatically with no setup]."

**What a weak positioning statement looks like:**
> "For businesses who want to grow, we are a software platform that helps them do better. Unlike our competitors, we have better features and better service."

(Too vague. No specificity on who, what problem, or why we win.)

### Value Prop Canvas: Red Flags

Watch for these during facilitation:

- Features without benefits -- translate to customer outcomes ("we automate X" --> "you get X hours back per week")
- "We're like X but better" -- better at what, specifically?
- Too many differentiators -- focus on ONE thing that matters most to the ICP
- No proof points -- claims without any evidence (even anecdotal) fall flat
- Focused on the company, not the customer -- flip the perspective
